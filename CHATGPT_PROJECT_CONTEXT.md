# MMing 个人网站项目说明

这份文档是给 ChatGPT 或其他 AI 助手看的，用来帮助它快速理解当前项目，避免误改项目方向。

## 项目简介

这是 MMing 的个人主页 / 个人博客网站。

网站主要用来展示：

- 个人介绍
- 正在学习的技能
- 学习项目
- 博客文章
- 联系方式

MMing 是一名正在学习 AI 和编程的学生。这个网站不是工作简历，也不要写虚假的工作经验。网站内容应该保持自然、真实，适合学生个人展示和学习记录。

## 技术要求

这个项目只使用：

- HTML
- CSS
- JavaScript

不要引入：

- React
- Vue
- Next.js
- Node 后端
- 数据库
- 复杂构建工具

原因：这个项目的目标是帮助新手理解前端基础，所以结构要简单，打开 `index.html` 就能预览。

## 主要文件说明

### `index.html`

网站首页，包含：

- 顶部导航栏
- 第一屏个人介绍
- 头像占位区域
- 关于我
- 我的技能
- 我的项目
- 博客文章卡片
- 联系方式
- 页脚版权信息

### `style.css`

网站样式文件，负责：

- 页面整体颜色
- 卡片样式
- 圆角和阴影
- 手机端适配
- 浅色 / 深色模式外观

### `script.js`

网站交互文件，目前主要负责：

- 深色模式切换
- 记住用户上一次选择的主题

### `.nojekyll`

给 GitHub Pages 使用的空文件，表示这个项目按普通静态网站发布，不使用 Jekyll 处理。

### `README.md`

给用户阅读的项目说明，包含项目介绍、文件结构、本地预览方法和维护建议。

### `posts/`

博客文章页面所在文件夹。

当前有 3 篇文章：

- `my-first-post.html`：我的第一篇博客
- `learning-ai.html`：我正在学习 AI
- `about-this-site.html`：关于这个网站

## 维护原则

以后修改这个项目时，请遵守这些原则：

1. 修改前先用通俗中文说明计划。
2. 修改后说明改了哪些文件。
3. 不要使用复杂框架。
4. 不要添加用户没有提供的隐私信息。
5. 不要写虚假的工作经历、奖项或项目成果。
6. 文案要自然一点，不要夸张。
7. 代码结构要适合新手理解。
8. 每次修改后检查页面链接是否正确。
9. 每次修改后检查手机端是否有横向溢出。
10. 保持首页和文章页的导航、样式、深色模式一致。

## 修改时重点检查

如果修改首页或文章页，请检查：

- `style.css` 路径是否正确。
- `script.js` 路径是否正确。
- 首页博客卡片是否能打开文章页。
- 文章页是否能返回首页。
- 邮箱链接是否仍然是 `mailto:` 格式。
- GitHub 链接是否能正常跳转。
- 手机屏幕下文字、按钮和卡片是否没有挤在一起。
- 深色模式按钮是否仍然可以点击。

## 给 ChatGPT 的提示

如果用户让你继续开发这个项目，请优先使用简单方案。

比如：

- 想新增文章：复制一份 `posts/` 里的文章页面，再修改标题和正文。
- 想新增技能：在首页技能区域增加一个卡片。
- 想新增项目：在首页项目区域增加一个卡片。
- 想改颜色：优先修改 `style.css` 顶部的颜色变量。
- 想改个人信息：优先修改 `index.html`，再同步检查文章页标题和页脚。

不要把这个项目改成复杂工程。这个网站最重要的是：简单、清楚、容易学习、容易发布。
